# 使用者訪談紀錄

> 這份文件由 Codex 依 Release 01 interview kit 產生草稿，使用三個 sub-agent 模擬 persona 回饋。這不是實際使用者訪談結果，學員必須刪除、修改或補充後才能當成決策依據。

## 我的 Phase 0 prototype 現況

- 首頁目前有三個主要入口：查詢頁面、工作人員頁面、災民上傳頁面。
- 查詢頁面會呈現 Phase 0 原始資訊、人工標籤、查核狀態，以及不能直接變成任務的線索。
- 工作人員頁面可以對原始資訊或上傳草稿加上人工整理標示，但這些標示不代表資料已確認。
- 災民上傳頁面只建立前端暫存草稿，並送到人工查核收件匣；它不是正式收案，也不會呼叫後端。

## Persona 1：回報者

### 我想完成的事

- 我想先把我知道的狀況送出去，就算地點、數量或需求還不完整。
- 如果我是轉述別人的話，我希望系統保留這件事，不要看起來像我親眼確認。

### 我看不懂的地方

- 「建立待審核草稿」和「送交人工查核」差在哪裡，我可能不確定送出後到底有沒有成功。
- 「已人工審核」我會有點怕被理解成「內容已確認是真的」。

### 我最怕的錯誤

- 我填了模糊線索，系統或後面的人把它當成確定地點。
- 我只是聽別人說，結果被當成現場本人回報。

### 我不敢直接相信的資訊

- 轉述來的需求、地點、時間，我都不敢保證完全正確。
- 上傳檔名或照片本身也不一定能證明狀況是真的，可能需要人再確認。

### 我希望系統提醒我的事

- 提醒我「不確定也可以說不確定」，不要逼我亂填。
- 提醒我不要放真實電話、完整地址或個資，也提醒我的回報還需要人工確認。

### 我想反問開發者的一個問題

- 我送出後，後面的人會怎麼知道哪些內容是我確定的、哪些只是聽說或猜的？

## Persona 2：資訊整理者

### 我想完成的事

- 我想把每筆原始資訊整理到下一位協作者看得懂：來源、目前狀態、哪些地方還不能判斷。
- 我想保留「原文」和「人工判斷」的差異，不希望整理後看起來像已經查證完。

### 我看不懂的地方

- 「已人工審核」和「已確認」的差別，我需要畫面一直提醒，否則很容易誤會。
- 「災民上傳頁面」送出的資料是模擬草稿還是真正進入資料清單，我一開始可能會搞混。

### 我最怕的錯誤

- 模糊地點、時間不明、來源未確認的資訊，被下一位協作者直接拿去派任務。
- AI 或系統自動補出原文沒有的內容，後面的人卻以為那是原始資訊。

### 我不敢直接相信的資訊

- 來自社群貼文、轉述、口述但沒有清楚來源脈絡的內容。
- 被人工加上分類標籤的資料，因為那只代表有人整理過，不代表內容是真的。

### 我希望系統提醒我的事

- 這批資料仍是 Phase 0 原始資訊，不是正式整理後資料。
- 哪些內容「需要人工確認」「不能直接相信」「不能直接變成任務」，不要只藏在細節裡。

### 我想反問開發者的一個問題

- 畫面上的每個狀態，下一位協作者會不會清楚知道它代表「整理狀態」還是「查證結果」？

## Persona 3：行動者

### 我想完成的事

- 我想快速分辨哪些資訊現在可以拿來決定下一步，哪些只是還在整理或等待確認。
- 我想知道自己如果要前往現場，地點、需求、狀態是不是清楚到足以行動。
- 我想避免因為畫面看起來很像「已處理完成」就誤以為可以直接出發。

### 我看不懂的地方

- 「已人工審核」和「已確認」對我來說很容易混在一起；我不確定人工看過是不是代表內容可信。
- 「災民上傳頁面」這個名稱會讓我以為資料比較接近當事人，但我不知道它是否仍需要查核。
- 我看不出一筆資訊是不是已經過期，或有沒有被後來的資訊推翻。

### 我最怕的錯誤

- 把 `needs_review` 或未確認資料看成可以直接行動，結果跑錯地方或造成現場負擔。
- 任務其實不存在了、需求改變了，但畫面沒有讓我停下來確認。
- AI 或整理者把模糊描述整理得太像事實，讓我誤判風險。

### 我不敢直接相信的資訊

- 只有社群貼文、轉述、口述來源的內容，我不敢直接當成現場狀況。
- 地點不清楚、時間不明、需求描述很籠統的資訊，我不敢拿來安排行動。
- 被標成「人工審核」但沒有說明審核到什麼程度的資料，我仍然會保留懷疑。

### 我希望系統提醒我的事

- 這筆資訊是否仍未確認，且未確認不等於可以行動。
- 這筆資訊有哪些地方會讓我不能直接變成任務，例如地點、時間、來源或安全風險。
- 如果資料只是前端暫存或展示用，我希望畫面不要讓我誤會它已經正式收案。

### 我想反問開發者的一個問題

- 當我看到一筆資料時，你希望我把哪一個標示當成「先停下來確認，不要直接行動」的最明確訊號？

## 人類補充

- 已採用：把「建立待審核草稿」和「送交人工查核」合併成單一「送出上傳」動作，避免回報者不知道自己是否已送出。
- 已採用：上傳者可以自行加上需求標籤，地點線索不再提供「某某附近」這類模糊地點，改保留「不確定」與備註。
- 已採用：移除檔案 / 照片欄位，避免 Phase 0 放入真實照片、地址或個資。
- 保留安全邊界：上傳資料送出後可在查詢頁看到，但仍標示為尚未查核；不把它顯示成已確認。
- 待補：哪些回饋符合你們實際操作 Phase 0 prototype 的感受？
- 待補：哪些 sub-agent 回饋太像 AI 推論，還需要真人訪談或小隊討論確認？
- 待補：你們是否同意「已人工審核」是目前最容易誤導的標示？
